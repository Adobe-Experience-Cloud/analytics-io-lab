Overview

The endpoints described here are routed through Adobe.io, so in order to use these endpoints your IMS client key MUST be subscribed to access the Adobe Analytics Reporting API.

Responses.

When an API request results in an error the response will contain both a response HTTP code and a response body describing the error.

Error Responses.

All errors return a JSON response in the following format:

{
    "errorCode": "invalid_data",
    "errorDescription": "Cannot create calculated metric. Invalid calculatedMetricId=213123sdf",
    "errorId": "cb2597f4-b790-4943-802a-a7e88f622a68"
}

The definition of the fields is as follows:

Field Description

errorCode

The type of error that occurred.

errorDescription

A description of the error not intented for human consupmtion. Usefull for troubleshooting the request. It tells you what was the problem with the request.

errorId

A GUID identifying the error. Useful to locate the stack trace within the logs if the errorDescription is not enough.

HTTP Error Codes

While there are many types of errors, HTTP response codes in the 4XX indicate a problem with the client request, in other words the problem is most likely with the client request. Http response codes in the 5XX range indicate a problem in the server.

Examples:

Response Headers.

There are additional response headers that can be used to troubleshoot the requests. x-request-id for example is a GUID that identifies the request, all logging in the services is tagged with the request id, so the logs can be searched even going across other services.

Reports

Ranked reports service

Runs a report for the request in the post body

POST /reports
Description

Runs a report using the criteria in given request post body

The /ranked endpoint provides access to your site’s analytics data. This is the same endpoint that the Analysis Workspace UI uses to get its data.

If you’ve used the Analytics APIs in the past, you’ll find some commonalities with that API, but will also notice some stark differences. The biggest difference you’ll notice is that breakdowns aren’t automatically requested for you. Requests need to be chained together retrieve breakdown data.

Best Practices

  • Use short queries that can be glued together. For example, don’t request 37 months of data with 20 metrics and 100 filters. Break that up into multiple requests. Its okay to request those smaller reports in parallel.

Overtime

Anomaly Detection
